
# Ballot Stamp Verifier

A production‑ready, offline‑capable pipeline to detect and validate official ballot stamps from scanned pages. Built for robustness (very low false positives), auditability, and speed on an **RTX 5060 Mobile** laptop GPU.

---

## 1) System Requirements

- OS: Windows 11 / Ubuntu 22.04+
- NVIDIA GPU: RTX 5060 Mobile (Ada/Lovelace)
- NVIDIA Driver: supports CUDA 12.x
- Python 3.10+

### 1.1 Install via `pip` (recommended)

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
.venv\Scripts\Activate.ps1

pip install --upgrade pip wheel setuptools
# Install PyTorch CUDA 12.1 wheels
pip install --index-url https://download.pytorch.org/whl/cu121 torch torchvision torchaudio

# Project deps
pip install -r requirements.txt

#Nvidia gpu
pip uninstall -y torch torchvision torchaudio
pip install --pre --upgrade --force-reinstall torch torchvision --index-url https://download.pytorch.org/whl/nightly/cu130
```

### 1.2 Install via Conda (alternative)

```bash
conda env create -f environment.yml
conda activate ballot-stamp
```

> The PyTorch cu121 wheels bundle the CUDA runtime, so you do **not** need a full local CUDA toolkit.

## 3) End‑to‑End Steps (Detailed)

### Step 0 — Prepare inputs
1. Collect a set of **blank ballot scans** (no stamps) in `data/blanks/`. Prefer diverse scanners, lighting, minor creases, and different page sizes (A4/Letter).
2. Place your official **transparent** stamp PNG (with alpha channel) at `assets/Stamp.png`.

### Step 1 — Generate synthetic training data
```powershell
python src/tools/generate_synthetic.py `
  --blanks_dir data/blanks `
  --stamp_png assets/Stamp.png `
  --out_dir data/synth `
  --count 6000 `
  --negative_ratio 0.30 `
  --decoy_ratio 0.70 `
  --placement mixed
```

What you get:
- `data/synth/images/` composited pages
- `data/synth/masks/` binary stamp masks
- `data/synth/labels_det/` YOLO detection labels
- `data/synth/labels_seg/` YOLO‑Seg placeholder polygons (optional)
- `data/synth/manifest.csv` presence metadata

The generator now creates a configurable mix of stamped and blank negative pages
(`--negative_ratio`, default `0.30`) and draws non-stamp decoys such as vote
numbers, dots, and strokes (`--decoy_ratio`, default `0.70`). Keep those enabled
when retraining; the detector needs empty-label ballots to learn that candidate
photos, party logos, handwriting, and the blank ballot template are not stamps.
With the current 3 blank ballot scans, 4,000 to 8,000 synthetic images is the
useful range. The command above uses 6,000 as a middle ground; if false positives
persist after retraining, increase to 8,000 before changing the model size.

Split into YOLO train/val/test:
```powershell
python src/tools/split_yolo.py `
  --images data/synth/images `
  --labels_det data/synth/labels_det `
  --out_dir data/yolo_split
```

### Step 2 — Train the detector (YOLO, CUDA 12.1)
```powershell
# Lightweight model for fast training/inference
python -m src.models.train_yolo --epochs 80 --imgsz 1024 --batch 16
```
Output weights:
- `runs/train/yolo_stamp/weights/best.pt`
- plus ONNX and TorchScript exports for offline use.

**Tips**
- If recall on faint stamps is low, switch to `yolov8s.pt` by adding `--model yolov8s.pt`.
- To speed up, reduce `--imgsz` to 1024.

### Step 3 — Inference with geometry validation + audit
```powershell
python -m src.infer.predict `
  --weights runs/train/yolo_stamp/weights/best.pt `
  --images data/synth/images `
  --out_dir outputs
```
Outputs per image:
- Annotated preview with decision label.
- Cropped candidate region (if any).
- `outputs/inference_manifest.json` with: decision, confidence, bbox, features (circularity, ellipse fit, template NCC), and skew angle.

Create an auditable roll‑up with model hash:
```powershell
python -m src.runtime.audit `
  --manifest outputs/inference_manifest.json `
  --weights runs/train/yolo_stamp/weights/best.pt
```

### Step 4 — Evaluate & calibrate thresholds
If you have ground‑truth YOLO labels for a validation set:
```powershell
python -m src.eval.evaluate `
  --pred_manifest outputs/inference_manifest.json `
  --labels_dir data/yolo_split/val/labels
```
Then sweep to meet a target false‑positive rate:
```powershell
python -m src.eval.calibrate_thresholds
```
Adjust `configs/val_rules.yaml`:
- `score_valid` ↑ to reduce false positives
- `score_review` sets **REVIEW REQUIRED** band
- Area and shape thresholds filter pen strokes/creases

### Step 5 — Batch pipeline / CLI
```powershell
# Generate + split + train + predict;
python -m src.runtime.service_cli --prepare_synth
python -m src.runtime.service_cli --train
python -m src.runtime.service_cli --predict --images data/SampleBallots
```

## 4) Current Two-Stamp + Real-Ballot Workflow

Use this workflow when the detector must learn both stamp assets:

- `assets/Stamp.png`
- `assets/stamp_2.png`

The working plan is:

1. Train a synthetic model with both stamp types.
2. Scan or photograph the 50 real printed ballots.
3. Label the real scanned ballots.
4. Fine-tune with 30 real train ballots plus synthetic data.
5. Validate on 10 real validation ballots.
6. Final-test once on 10 untouched real test ballots.

Run these commands from the `stamp-detection` folder.

### 4.1 Back up the current single-stamp model

If the current `best.pt` is the 6,000-ballot model trained only on `Stamp.png`, save it before retraining:

```powershell

```

If `runs\train\yolo_stamp\weights\best.pt` has already been overwritten, use the existing backup:

```text
runs\train\best_synth6000_stamp1_only.pt
```

### 4.6 Add and split the real scanned ballots

Put scanned or photographed real ballots here:

```text
data/real_scans/train/images
data/real_scans/train/labels
data/real_scans/val/images
data/real_scans/val/labels
data/real_scans/test/images
data/real_scans/test/labels
```

Use this 50-ballot split:

| Split | Stamp 1 | Stamp 2 | No stamp | Total |
|---|---:|---:|---:|---:|
| Train | 12 | 12 | 6 | 30 |
| Val | 4 | 4 | 2 | 10 |
| Test | 4 | 4 | 2 | 10 |

Each physical ballot must appear in one split only. Do not put the same image, or an edited copy of the same physical ballot, in both train and validation/test.

### 4.7 Label real ballots with the local web labeler

`labelImg` can crash on some Python/PyQt versions with float/int Qt errors. Use the local browser labeler instead.

Train split:

```powershell
python .\src\tools\label_yolo_web.py `
  --images .\data\real_scans\train\images `
  --labels .\data\real_scans\train\labels `
  --port 8765
```

Open:

```text
http://127.0.0.1:8765
```

For each image:

- Draw one tight box around the visible stamp and click **Save Box**.
- For no-stamp ballots, click **No Stamp**.
- Both `Stamp.png` and `stamp_2.png` use the same class: `stamp`.
- For partly visible stamps, box only the visible stamp area.

Keyboard shortcuts:

- `D`: next image
- `A`: previous image
- `S`: save box
- `N`: no stamp
- `Delete`: clear current box

Stop the server with `Ctrl+C`, then label validation:

```powershell
python .\src\tools\label_yolo_web.py `
  --images .\data\real_scans\val\images `
  --labels .\data\real_scans\val\labels `
  --port 8765
```

Then label test:

```powershell
python .\src\tools\label_yolo_web.py `
  --images .\data\real_scans\test\images `
  --labels .\data\real_scans\test\labels `
  --port 8765
```

Check that every image has a matching label file. This command should print nothing:

```powershell
Get-ChildItem .\data\real_scans\train\images\*.* | ForEach-Object {
  $label = ".\data\real_scans\train\labels\$($_.BaseName).txt"
  if (!(Test-Path $label)) { $_.Name }

```

Repeat the same check for `val` and `test`.

### 4.8 Fine-tune with real train ballots plus synthetic data

Create image lists. The real training images are repeated 25 times so the 30 real ballots matter during fine-tuning.

```powershell
New-Item -ItemType Directory -Force .\data\lists | Out-Null

$synth = @()
$synth += Get-ChildItem .\data\yolo_split_stamp1\train\images\*.png
$synth += Get-ChildItem .\data\yolo_split_stamp2\train\images\*.png
$synth = $synth | ForEach-Object { $_.FullName.Replace('\','/') }

$realTrain = Get-ChildItem .\data\real_scans\train\images\*.* |
  Where-Object { $_.Extension -match '\.(jpg|jpeg|png|tif|tiff)$' } |
  ForEach-Object { $_.FullName.Replace('\','/') }

$realRepeated = 1..25 | ForEach-Object { $realTrain }

$synth + $realRepeated | Set-Content .\data\lists\train_mixed.txt
```

```powershell
Get-ChildItem .\data\real_scans\val\images\*.* |
  Where-Object { $_.Extension -match '\.(jpg|jpeg|png|tif|tiff)$' } |
  ForEach-Object { $_.FullName.Replace('\','/') } |
  Set-Content .\data\lists\val_real.txt
```

```powershell
Get-ChildItem .\data\real_scans\test\images\*.* |
  Where-Object { $_.Extension -match '\.(jpg|jpeg|png|tif|tiff)$' } |
  ForEach-Object { $_.FullName.Replace('\','/') } |
  Set-Content .\data\lists\test_real.txt
```

Create the fine-tune dataset config:

```powershell
@'
path: .
train: data/lists/train_mixed.txt
val: data/lists/val_real.txt
test: data/lists/test_real.txt

names:
  0: stamp
'@ | Set-Content .\configs\yolo_dataset_real_finetune.yaml
```

Fine-tune from the two-stamp synthetic model:

```powershell
python -m src.models.train_yolo `
  --data .\configs\yolo_dataset_real_finetune.yaml `
  --model .\runs\train\best_synth_both_stamps.pt `
  --epochs 30 `
  --imgsz 1024 `
  --batch 16
```

Save the real fine-tuned model:

```powershell
Copy-Item .\runs\train\yolo_stamp\weights\best.pt .\runs\train\best_real_finetuned.pt
```

## 5) Design Rationale

- **Detector-first + geometry**: Higher recall on faint/partial stamps; shape/area/template checks suppress FP from pen strokes, logos, folds, smudges.
- **No fixed location**: Model scans whole page; `deskew.py` normalizes skew; CLAHE mitigates lighting.
- **Auditability**: Crops, overlays, scores, features, model hash, and an audit JSON for every decision.
- **Offline**: No network calls; ONNX/TorchScript exports available.

## 6) Troubleshooting

- **Torch says no CUDA**: Ensure NVIDIA driver is up-to-date; reinstall cu121 wheels as shown above.
- **Ultralytics import error**: `pip install ultralytics==8.3.6` in the same environment.
- **Poor recall on faint stamps**: Increase training images, lower detector conf during predict (code default `--conf 0.1`), or try `yolov8s.pt`.
- **Too many false positives**: Raise `score_valid`, tighten `min_circularity` or `min_ellipse_edge_pct`, add more hard negatives.
- **`labelImg` crashes with PyQt float/int errors**: Use `python .\src\tools\label_yolo_web.py --images <images_dir> --labels <labels_dir> --port 8765` and open `http://127.0.0.1:8765`.
